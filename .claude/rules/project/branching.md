# Branching Rule

Never commit directly to `main`. Before making any changes, create a feature branch and work on it.

```bash
git checkout -b <type>/<short-description>
```

| Type | Example |
|------|---------|
| `feat/` | `feat/mini-player` |
| `fix/` | `fix/build-failure` |
| `refactor/` | `refactor/player-state` |
| `chore/` | `chore/deps-update` |
| `docs/` | `docs/schema-descriptions` |
