# Communication Language Guidelines

## Git & GitHub & jj(jujutsu)

| Context | Language |
|---------|----------|
| Commit messages | English |
| Pull request titles & descriptions | English |
| Issue titles & descriptions | English |
| Code review comments | English |
| GitHub discussions | English |

## Codebase

| Context | Language |
|---------|----------|
| Code comments | English |
| Documentation files | English |
| API documentation | English |
| Error messages in code | English |

## Claude Communication

Respond in the user's device default language (or the language the user writes in).

## Commit Message Format

Use conventional commits:
- `feat(scope):` New features
- `fix(scope):` Bug fixes
- `refactor(scope):` Code refactoring
- `docs(scope):` Documentation
- `test(scope):` Tests
- `chore(scope):` Maintenance

## Pull Request Format

```markdown
## Summary
- Bullet points describing changes
- Focus on "what" and "why"

## Test plan
- [ ] Verification steps
```
